# 에버소울 오프라인 백엔드 서버 종합 분석 및 레거시 버그 리포트 (Consolidated Issue)

이 문서는 기존 `docs/report/` 디렉터리에 분산되어 있던 버그 분석, 백엔드 아키텍처, 그리고 C++ 코딩 스타일 가이드를 단일 이슈(Issue) 문서로 완벽하게 통합 및 정리한 버전입니다. 프로젝트의 기술적 기반과 해결해야 할 레거시 결함들을 총망라합니다.

---

## 1. 레거시 버그 리포트 및 런타임 디버깅 가이드

본 섹션은 0.0.3 버전 등에서 발생한 핵심 소프트락/버그에 대한 상세 분석 내용과, 항상 연결된 ADB 환경을 활용하여 인게임 런타임 동작(TBL, 번들 로드, 글로벌 메타 등)을 실시간으로 디버깅하는 방법을 정리합니다.

### 1.1. 피드백 수집 버그 분석 리포트
* **전투/DJ소울/연습실 이벤트 진입·클리어 시 무한 로딩 (소프트락)**
  * **원인**: DJ소울 라우팅 테이블 누락(404 에러), 레이드 액션 엔드포인트 누락.
* **캐릭터 잠재능력(Zodiac) 진입 시 소프트락**
  * **원인**: `/ZodiacPresetList` 상태형 엔드포인트 미등록, `ZodiacActive` 빈 응답(stub) 처리.
* **출석 체크 인식 불가로 인한 무한 팝업 루프**
  * **원인**: `/AttendanceInfo` 엔드포인트가 빈 응답으로 처리되어 클라이언트가 출석 완료를 인지하지 못함.
* **신규 Guest 계정 생성 시 기존 sensei 계정으로 강제 연동되는 버그**
  * **원인**: `/Login` 라우터에서 하드코딩된 `playerId`(`431921183232`)를 가진 `Login.json` 픽스쳐를 맹목적으로 반환하여 세션이 오염됨.

### 1.2. ADB 기반 런타임 디버깅 가이드
* 인게임 데이터는 모두 복호화되어 동작하므로, ADB 9991 포트 포워딩 후 로그캣(`logcat`)을 통해 실시간 Unity/C# 로그를 수집합니다.
* **명령어**: `adb logcat -s Unity | grep -iE "network|error|exception|addressables|gamedata|tbl"`
* **디버깅 대상**: 인게임 TBL 로드 추적, Addressables 에셋 번들 로딩 구조(`/Live/`), 카카오 SDK 글로벌 메타 인증 흐름 추적.

## 2. 백엔드 시스템 아키텍처 분석

오프라인 서버 내부의 인증 방식, TBL 런타임 데이터 매핑, 에셋/카탈로그 번들 로딩 및 글로벌 메타 정보 제어 흐름에 대한 C++ 시스템 엔지니어링 분석입니다.

### 2.1. 백엔드 인증 방식 및 로그인 흐름
* **카카오 SDK 장치 토큰 생성 (`accessToken/create`)**: 서버가 자체적으로 `offline-zat-` 토큰을 생성.
* **이용약관 동의 및 자동 로그인 (`agreement/getForLogin`)**: `zat` 감지 시 즉시 약관 동의 상태로 우회 처리.
* **디바이스 게스트 인증 (`loginDevice`)**: 구글/페북 로그인을 게스트 프로토콜(`zd3`)로 강제 바이패스시켜 인증 메타 생성.
* **실물 게임 로그인 (`/Login`)**: Protobuf 기반 통신으로 세션 발급. 픽스쳐 또는 동적 페이로드 반환.

### 2.2. TBL 로딩 및 런타임 데이터 연계 구조
* **TblStore (로더)**: 서버 시작 시 359개 TBL을 메모리에 파싱하고 `no` 필드를 인덱싱 키로 삼아 $O(1)$ 탐색을 보장.
* **런타임 데이터 합성**: 소울 시그니처 정령 리스트 등 클라이언트가 대형 리스트를 요구할 때, 정적 픽스쳐가 아닌 메모리의 TBL과 유저의 AccountDB 상태를 결합하여 실시간 Protobuf 직렬화를 수행.

### 2.3. 글로벌 메타 정보 및 CDN 제어
* 카카오 인포데스크(`/v2/app`) 응답을 통해 cdnAddr과 gameServerAddr을 로컬호스트(`127.0.0.1:9991`)로 강제 리다이렉트.
* `/Live/` 라우터를 통해 로컬 캐시 폴더에 존재하는 카탈로그 및 번들 파일을 스트리밍 서빙.

### 2.4. 실시간 런타임 디버깅 필터링 시스템
* `logcat_process.cpp`에서 에뮬레이터 로그 스트림을 필터링하여 TBL QUERY, BUNDLE/CATALOG, GLOBAL META 이벤트를 추출.
* ANSI 컬러를 입혀 터미널에 시각화하고 `runtime_debug.log`에 영속적으로 덤프 처리함.

## 3. C++ 코딩 스타일 및 아키텍처 패턴

C++ 백엔드 서버 소스 코드들의 전반적인 코딩 표준, 아키텍처 패턴, 오류 처리 철학 규칙입니다. 본 프로젝트 개발 시 절대적으로 준수해야 합니다.

### 3.1. 코드 포맷팅 및 명명 규칙
* **들여쓰기**: 4칸 공백 (하드 탭 금지). `namespace` 블록 내부는 들여쓰지 않음.
* **중괄호 배치**: Allman Style (다음 줄에 여는 중괄호). 단, one-liner 루프/if는 중괄호 생략 허용.
* **명명 규칙**:
  * Namespace / Method / Local Var: `snake_case`
  * Class / Struct: `PascalCase`
  * Private Member: `snake_case_` (언더바 접미사)
  * Constant / Enum Prefix: `kCamelCase`

### 3.2. 에러 및 예외 처리
* **`try-catch` 및 예외 사용 최소화**: 함수의 반환형(Return Type)을 통해 성공/에러 상태를 관리합니다.
* **`[[nodiscard]]` 강제화**: 상태 확인이 필수적인 반환값은 무시할 수 없도록 컴파일러 레벨에서 경고.
* **`std::optional<T>` 적극 활용**: 데이터 룩업 실패 시 널포인터나 예외 대신 사용.

### 3.3. 네임스페이스 및 메모리 관리
* **익명 네임스페이스(`namespace { ... }`)**: .cpp 내부 전용 헬퍼 함수 및 정적 변수에 강제 적용하여 링킹 충돌(Internal Linkage)을 방지.
* **Value Semantics (값 복사 체계)**: `new/delete` 및 스마트 포인터를 배제하고 구조체 값(Value) 자체를 복사/이동하여 전달해 메모리 누수를 원천 차단.
* **자체 JSON 파서(`eversoul::json`) 사용**: `int64_t` 정밀도 손실 방지를 위해 무거운 외부 라이브러리(`rapidjson` 등)를 지양.
* **C 스타일 DTO 초기화**: 구조체 멤버 변수 선언 시 인라인으로 `= 0` 또는 기본값을 할당하여 쓰레기값 방지.
